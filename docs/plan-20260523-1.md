# 세포 키우기. Grow your cell - 2026-05-23 기획/구현 계획

## 제품 방향

게임 이름은 `세포 키우기. Grow your cell`로 확정한다.

화면에서는 로고 가독성을 위해 다음처럼 2단 구성을 우선한다.

```text
세포 키우기
Grow your cell
```

핵심 감각은 "현미경으로 본 진짜 세포를 게임용으로 정리한 생존 성장 게임"이다. 너무 현실적인 점액감이나 혐오감을 주는 묘사는 피하고, 형태와 움직임은 실제 세포에서 가져오되 색감과 피드백은 명확한 게임 표현을 따른다.

## 조작/AI 원칙

자동 이동은 방치형 자동 플레이가 아니라 세포의 본능적 행동을 플레이어가 조율하는 방식이어야 한다.

특히 모바일에서는 지속적인 방향 조작이 부담이므로 자동 이동을 기본값으로 둘 수 있어야 한다. PC에서는 키보드 직접 조작이 손쉬우므로 수동 조작을 기본으로 두되, AI 조율 모드를 선택할 수 있어야 한다.

### 이동 모드

- `manual`: 키보드 입력 기반 직접 조작.
- `auto`: 세포가 점수 함수에 따라 자동으로 이동.

### 자동 이동 점수 함수

자동 이동은 다음 평가축을 사용한다.

- 먹이 가치.
- 내가 먹을 수 있는 적의 가치.
- 나를 먹을 수 있는 적의 위험.
- 염분 구역 회피.
- 소용돌이 위험 또는 활용.
- 분열 세포 재결합 욕구.
- 현재 질량, 속도, 스킬 상태에 따른 리스크.

사용자 인터랙션은 방향 명령보다 우선순위 조율에 가깝게 설계한다.

- 성장 우선.
- 생존 우선.
- 사냥 우선.
- 위험 감수.
- 재결합 우선.
- 특정 위치/대상 관심 지정.
- 스킬 발동 타이밍 선택.

## 세포 표현

세포막은 단독 생물처럼 움직이면 안 된다. 이동 판단은 `Player` 또는 이동 브레인이 담당하고, 세포막/개별 세포 조각은 물리와 상태에 반응하는 표현 레이어여야 한다.

### 시각 방향

- 얇은 반투명 이중 세포막.
- 세포질 내부 노이즈와 작은 과립.
- 불규칙하지만 귀엽고 명확한 핵.
- 핵소체 느낌의 작은 내부 점.
- 플레이어, 적, 먹이, 펫이 멀리서도 구분되는 색상 유지.
- 포자 모드는 딱딱한 껍질과 수축된 세포질 느낌.

### 폰트 방향

- 타이틀/로고: `Bagel Fat One`.
- 한국어 UI/본문: `Noto Sans KR`.
- 영어/숫자 보조 UI: `Outfit`.

Godot 프로젝트에서는 가능하면 폰트 파일을 프로젝트에 포함하고 UI에 연결한다. 웹 프로토타입은 Google Fonts 링크를 사용할 수 있다.

## 섭취/식세포 작용 이펙트

현재 충돌 즉시 질량 증가 후 대상이 사라지는 방식은 충돌 삭제처럼 보인다. 섭취는 짧은 연출 상태를 거쳐야 한다.

### 섭취 흐름

1. 먹을 수 있는 대상과 접촉.
2. 대상이 `absorbing` 상태로 전환되고 충돌/AI가 비활성화됨.
3. 먹는 세포의 접촉 방향 세포막이 밝아지고 움푹 파임.
4. 대상 주변에 포획막/파동이 생김.
5. 대상이 먹는 세포 중심으로 끌려가며 작아지고 투명해짐.
6. 내부 소화 파동/입자 이펙트.
7. 완료 시 질량/점수 반영 후 대상 제거.

### 우선 적용 순서

1. 플레이어가 먹이를 먹을 때.
2. 플레이어가 적을 먹을 때.
3. 적이 플레이어를 먹을 때.
4. 적이 먹이/다른 적을 먹을 때.

1차 구현에서는 플레이어가 먹이와 적을 먹는 경우를 우선한다.

## 1차 구현 범위

- 프로젝트 이름과 UI 제목을 `세포 키우기. Grow your cell` 방향으로 정리.
- `manual`/`auto` 이동 모드와 자동 이동 점수 함수 1차 구현.
- `Tab` 또는 `M`으로 이동 모드 전환.
- 자동 이동 기본 우선순위는 `growth`로 시작하고, 이후 모바일 UI에서 성향 선택으로 확장 가능하게 만든다.
- `cell_shader.gdshader`에 현미경풍 세포막/세포질/과립/핵소체 표현 추가.
- 섭취 대상이 즉시 사라지지 않고 짧은 흡수 연출 후 보상이 반영되도록 1차 구현.

## 구현 리스크

- 모든 충돌을 비동기 흡수 상태로 바꾸면 중복 섭취/중복 보상 문제가 생길 수 있다.
- 분열 세포가 여러 개일 때 같은 먹이를 동시에 흡수하지 않도록 `is_absorbing` 플래그가 필요하다.
- 자동 이동 AI가 너무 잘하면 방치형처럼 보이고, 너무 못하면 답답하다. 기본은 보수적인 성장/회피로 두고 조율 축을 확장한다.
- 셰이더 디테일은 먹이 수가 많으므로 과도하면 성능을 해칠 수 있다.

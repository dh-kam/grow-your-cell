# 2026-05-24 기획 및 적용 기록: 암세포, 스테이지, 토큰 스토어

## 요청 요약

긴장감을 높이기 위해 게임 중간중간 암세포가 발생하도록 한다. 암세포는 접촉한 다른 세포를 파괴하거나 암세포로 감염시킨다.

우리 세포가 암세포와 접촉했을 때의 규칙:

- 보유 세포가 하나뿐이면 암세포 접촉 시 게임 종료.
- 분열 등으로 여러 세포 조각을 가진 상태라면 접촉한 세포 조각만 영향을 받는다.
- 영향을 받은 세포 조각은 확률에 따라 암세포화되거나 터져 사라진다.

진행 구조:

- 스테이지 개념을 도입한다.
- 일정 크기까지 성장하면 스테이지 클리어 후 다음 스테이지로 진행한다.
- 스테이지가 높아질수록 암세포화 확률이 증가한다.
- 스테이지가 높아질수록 암세포 발생 확률도 증가한다.
- 리소좀 투사체로 암세포를 공격해 크기를 줄이거나 죽일 수 있다.

성장 보상 및 선택:

- 스테이지 클리어마다 토큰을 지급한다.
- 토큰으로 메인 메뉴의 스토어에서 다른 종류의 세포를 구매할 수 있다.
- 구매한 세포가 있다면 스테이지 시작마다 플레이할 세포를 선택할 수 있다.

## 적용된 설계

웹 프로토타입을 기준 구현으로 삼고, Godot 프로젝트에는 암세포와 스테이지의 핵심 규칙을 맞춰 반영했다.

### 스테이지

- 현재 스테이지, 목표 질량, 암세포 위험도, 보유 토큰을 점수 패널에 표시한다.
- 목표 질량은 스테이지가 오를수록 증가한다.
- 목표 질량에 도달하면 스테이지 클리어로 처리하고, 현재 스테이지 값만큼 토큰을 지급한다.
- 클리어 후 메인 메뉴로 돌아가 다음 스테이지 시작 전 세포 선택과 스토어 구매를 할 수 있다.

추가 규칙:

- 스테이지 목표 질량은 해당 스테이지에서 세포가 안정적으로 버틸 수 있는 안전 한계다.
- 정상 성장 처리에서는 목표 질량까지만 질량 증가를 적용하고, 그 지점에서 스테이지 클리어로 전환한다.
- 버그나 예외 경로로 총 세포 질량이 안전 한계를 초과한 상태가 감지되면 세포가 파열되며 게임 오버가 된다.
- 이 규칙 때문에 스테이지는 목표 질량을 넘어선 상태로 진행될 수 없다.

### 암세포

- 암세포는 보라색 세포로 표시한다.
- 암세포는 기존 적 세포 타입에 `cancer` 타입으로 추가했다.
- 스테이지가 오를수록 암세포 발생 확률과 동시 존재 가능한 암세포 수가 증가한다.
- 암세포는 플레이어 세포를 적극적으로 추적한다.
- 플레이어가 보이지 않을 때는 일반 적 세포 쪽으로 이동하며, 접촉한 적 세포를 감염시키거나 약화시킨다.
- 먹이를 접촉하면 암세포가 약간 성장하고, 일정 확률로 먹이 위치에 새 암세포가 생긴다.
- 화면 밖 근처의 암세포는 경고 인디케이터에 `CANCER`로 표시한다.

### 플레이어와 암세포 접촉

- 플레이어 세포가 1개뿐일 때 암세포와 접촉하면 즉시 게임 오버.
- 플레이어 세포가 여러 개일 때는 접촉한 조각만 제거된다.
- 제거된 조각은 확률에 따라 암세포로 변하거나, 그냥 터져 사라진다.
- 암세포화 확률은 스테이지가 높을수록 증가한다.
- 세포 타입별 암저항 값이 암세포화 확률을 낮춘다.

### 리소좀과 암세포

- 리소좀 투사체는 암세포 질량을 줄일 수 있다.
- 암세포는 일반 적보다 리소좀에 더 취약하게 처리했다.
- 암세포 질량이 낮아지면 제거된다.
- 대식 세포 타입은 리소좀 위력이 높아 암세포 대응에 유리하다.

### 세포 스토어

메인 메뉴에 스토어와 세포 선택 영역을 추가했다.

기본 보유:

- 기본 세포: 균형 잡힌 표준 세포.

토큰 구매 세포:

- 섬모 민첩 세포: 속도와 가속이 높고 시작 질량은 낮다.
- 대식 세포: 리소좀 위력이 높고 암저항이 있다. 움직임은 약간 둔하다.
- 막강화 세포: 암저항이 가장 높고 시작 질량이 높지만 이동성과 리소좀 위력이 낮다.

구매 상태와 선택 세포는 `localStorage`에 저장한다.

## 주요 적용 파일

- `web_prototype/index.html`
  - 스테이지 UI, 토큰 UI, 스토어 UI, 세포 선택 UI
  - 세포 타입 데이터와 구매/선택 저장
  - 암세포 생성, 감염, 파괴, 리소좀 피해, 경고 표시
  - AI 자동 이동의 암세포 회피 반영

- `enemy.gd`
  - Godot 적 타입에 암세포 타입 추가
  - 암세포 색상, 이동, 감염/약화 처리

- `main.gd`
  - Godot 스테이지, 토큰 카운트, 암세포 스폰 타이머, 목표 질량 표시
  - 게임 오버 처리 기본 문구

- `player.gd`
  - 암세포 접촉 시 단일 세포 게임 오버
  - 분열 상태에서는 접촉 세포만 제거 또는 암세포화
  - AI 자동 이동에서 암세포 강회피 반영

- `acid_projectile.gd`
  - 리소좀 투사체가 암세포를 더 강하게 줄이고, 낮은 질량의 암세포를 제거

- `main.tscn`
  - 도움말에 암세포 회피 및 리소좀 공격 안내 추가

## 검증

`cell-term` tmux 세션에서 다음을 확인했다.

- `git diff --check`
- `godot4 --headless --path . --quit-after 1`
- 웹 프로토타입 스모크 테스트
- 웹 메뉴/스토어/세포 선택 렌더링 스모크 테스트
- `http://127.0.0.1:8000/` 200 OK

`cell-run` tmux 세션은 기존대로 `web_prototype`을 8000번 포트에서 서빙 중이다.

## 남은 개선 후보

- Godot 쪽 메인 메뉴 스토어 UI는 아직 웹 기준 구현만큼 완성하지 않았다.
- 스테이지 클리어 연출은 현재 메뉴 복귀 중심이며, 별도 애니메이션/전환 연출을 추가할 수 있다.
- 암세포 감염/파괴 확률과 스폰 빈도는 실제 플레이 감각을 보면서 튜닝이 필요하다.
- 토큰 저장 초기화 버튼이나 디버그 패널이 있으면 테스트가 쉬워진다.
